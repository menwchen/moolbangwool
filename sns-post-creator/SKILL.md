# SNS Post Creator

주제를 받으면 리서치를 수행하고, 각 소셜미디어 플랫폼의 특성에 맞는 포스트를 자동으로 생성한다.

## 워크플로우

### 1단계: 주제 파악

사용자가 제공한 주제를 분석한다. 주제가 모호하면 간단히 확인 질문을 한다. 하지만 대부분의 경우 바로 리서치에 들어간다.

파악할 것:
- 핵심 주제/키워드
- 타겟 독자층 (명시되지 않으면 일반 대중으로 가정)
- 톤 (명시되지 않으면 각 플랫폼 기본 톤 사용)
- 언어 (사용자가 한국어로 요청하면 한국어, 영어면 영어)

### 2단계: 웹 리서치

WebSearch 도구로 주제에 대한 최신 정보를 수집한다. 리서치의 목적은 포스트에 신뢰성과 구체성을 더하는 것이다.

검색 전략:
- 주제 관련 최신 뉴스/트렌드 (2-3개 검색)
- 관련 통계나 데이터 포인트
- 현재 SNS에서 해당 주제가 어떻게 다뤄지고 있는지

리서치 결과에서 포스트에 활용할 수 있는 것들을 추출한다:
- 구체적인 숫자/통계
- 흥미로운 팩트
- 트렌딩 앵글이나 관점
- 관련 해시태그

### 3단계: 플랫폼별 포스트 생성

각 플랫폼의 문화와 알고리즘 특성을 반영하여 같은 주제를 다르게 풀어낸다. 단순히 길이만 조절하는 게 아니라, 플랫폼 고유의 커뮤니케이션 방식에 맞춘다.

---

#### X (Twitter)

**글자 제한**: 280자 (한국어 기준으로도 280자)
**핵심 특성**: 빠르고 날카롭다. 의견과 위트가 중요하다. RT와 인용RT를 유도하는 글이 좋다.

톤 가이드:
- 짧고 펀치력 있는 문장
- 하나의 핵심 메시지에 집중
- 의견이나 관점을 담아 반응을 유도
- "~인 거 알고 계셨나요?", "솔직히 말하면~" 같은 hook 활용
- 이모지는 1-2개만 포인트로 사용
- 해시태그 1-3개 (본문 끝에)

구조:
```
[Hook 문장 - 시선을 끄는 한 줄]
[핵심 메시지 - 구체적 팩트나 의견]
[해시태그]
```

나쁜 예: "오늘 흥미로운 기사를 읽었습니다. AI에 대한 내용이었는데요..."
좋은 예: "GPT-5 나오면 코딩 면접 사라질 수도. 이미 구글 내부에서 AI가 짠 코드 비율이 25% 넘었다는데 🤔 #AI #개발자"

---

#### Threads

**글자 제한**: 500자
**핵심 특성**: 인스타그램 생태계 기반. 캐주얼하고 친근한 대화체. 개인적인 이야기나 의견 공유가 잘 먹힌다. 텍스트 중심이지만 감성적인 접근이 효과적.

톤 가이드:
- 친구에게 말하듯 편한 말투 (하지만 반말은 아닌, "~하더라고요", "~인 것 같아요" 체)
- 개인적 경험이나 감상을 섞으면 좋다
- 줄바꿈을 활용해 가독성 확보
- 이모지를 자연스럽게 활용 (3-5개)
- 질문으로 끝나면 댓글 유도에 좋다
- 해시태그는 안 쓰거나 최소한으로 (Threads 문화)

구조:
```
[공감 포인트나 개인적 이야기로 시작]

[주제에 대한 본론 - 2-3문장]

[마무리 - 질문이나 생각 공유 요청]
```

나쁜 예: "[공지] AI 기술 동향 보고서에 따르면 생성형 AI 시장이..."
좋은 예: "요즘 AI로 코딩하는 거 보면서 느낀 건데요\n\n솔직히 처음엔 '이게 되나?' 싶었는데 써보니까 진짜 세상 편하더라고요 😮\n\n근데 한편으로는 이러다 개발자가 정말 필요 없어지는 거 아닌가 하는 생각도 들고...\n\n여러분은 AI 코딩 도구 써보셨나요? 어떠셨어요?"

---

#### Facebook

**글자 제한**: 사실상 제한 없음 (하지만 300-800자가 이상적)
**핵심 특성**: 스토리텔링과 정보 공유. 다른 플랫폼보다 긴 글이 용인된다. 공유를 유도하는 가치 있는 콘텐츠가 좋다. 연령대가 다양하므로 보편적 접근이 필요.

톤 가이드:
- "~합니다" 체의 정중하지만 딱딱하지 않은 톤
- 스토리텔링 구조 (도입-전개-결론)
- 구체적인 정보나 인사이트를 담아 공유 가치를 높인다
- 이모지는 포인트로 사용 (과하지 않게)
- 해시태그 3-5개 (본문 끝에)
- "공유해주세요", "여러분의 생각은?" 같은 CTA 가능

구조:
```
[관심을 끄는 도입부 - 왜 이 주제가 중요한지]

[본론 - 리서치 기반 구체적 정보 2-3개]

[개인적 의견이나 시사점]

[CTA - 공유 또는 의견 요청]

[해시태그]
```

### 4단계: 출력 형식

리서치 요약과 각 플랫폼별 포스트를 깔끔하게 정리해서 보여준다.

출력 구조:

```
## 📋 리서치 요약
[주제에 대해 찾은 핵심 정보 3-5줄 요약]

---

## 𝕏 X (Twitter)
[포스트 내용]
📏 글자 수: N자

---

## 🧵 Threads
[포스트 내용]
📏 글자 수: N자

---

## 📘 Facebook
[포스트 내용]
📏 글자 수: N자
```

### 추가 옵션

사용자가 요청하면 다음도 제공한다:
- **A/B 버전**: 각 플랫폼별 2가지 버전
- **영어 버전**: 한국어 포스트의 영어 번역 (단순 번역이 아닌, 영어권 SNS 문화에 맞게 재작성)
- **이미지 프롬프트**: 포스트에 어울리는 이미지 생성 프롬프트 (DALL-E/Midjourney용)
- **게시 시간 추천**: 각 플랫폼별 최적 게시 시간대

## 주의사항

- 리서치에서 얻은 정보의 출처를 리서치 요약에 명시한다
- 각 포스트가 해당 플랫폼의 글자 제한을 지키는지 확인한다
- 허위 정보나 검증되지 않은 통계는 사용하지 않는다
- 저작권이 있는 콘텐츠를 그대로 인용하지 않는다
- 민감한 주제(정치, 종교 등)는 중립적 톤을 유지한다
